## §未決

- **Toast surface**: no SudokuUI toast infrastructure exists yet — surface success / failure as an inline Form `Label` row (mirrors the existing `clearCacheConfirmation` pattern). Real toast is a future v2.4 cosmetic item.
- **Purchase event → state reactivity**: `IAPClient.purchaseUpdates()` (refunds / external transactions / family-share revocations) is *not yet* subscribed by the controller. Out-of-band entitlement loss would currently leave `hasPurchasedRemoveAds = true` until next bootstrap. Deferred to v2.4.
- **Boot-step telemetry**: `TelemetryEvent` is a strongly-typed catalog with no "info"/"breadcrumb" case; boot success is logged via `print(...)` and failures via `Telemetry.observe(.errorOccurred(source:"MonetizationBoot",code:step,...))`. Promoting boot events to a first-class `TelemetryEvent.monetizationBootOutcome(...)` case is a v2.4 cosmetic.
- **iPad / macOS Settings layout**: only iPhone-light snapshot baselines were added for the IAP rows; Mac + dark variants follow the same `Form` chrome and were not part of the v2.3.6 spec scope.
- **`removeAdsProductId` duplication**: declared as a top-level `public let` in SudokuUI mirroring `IAPStoreKit2.IAPProductIDs.removeAds`. The ASC product ID is the shared contract; duplicating one string is cheaper than expanding the MonetizationCore frozen surface.

## Decisions

1. **Single shared controller instance** — built in `AppComposition.live()` / `.fakeComposition()`, threaded through both `RootView → HomeView` and `LiveRouteFactory → SettingsView`. Both surfaces observe the same `@Observable` so a purchase tap on the Home card flips the Settings row visibility on the same frame.
2. **Boot coordinator uses closure seam** (`MonetizationBootBridges`) rather than exposing `UMPBridge` / `ATTBridge` publicly — keeps the internal seams internal (per spec "don't touch frozen MonetizationCore surface" + AdsAdMob private impls stay private).
3. **No new AppRoute or NavigationLink growth** — Remove Ads is a button inside HomeView (not a new mode card with subtree); 5th-card layout mirrors `ModeCard` so the LazyVGrid row metrics stay consistent.
