# v2.3 Part B — Banner UI wiring (v2.3.4 + v2.3.5)

Date: 2026-05-21
Branch: `feat/v2.3-part-b-banner-ui`
Scope: HomeView + BoardView banner slot mount.

## Summary

Threads `AdProvider` + `AdGate` from `AppComposition` → `RootView` →
`HomeView` (root mount) and `RootView` → `RouteFactory` → `BoardLoaderView`
→ `BoardView`. The new `BannerSlotView` consults `AdGate` on mount; when
the gate denies it collapses to `EmptyView()` (0pt). When the gate allows
it reserves exactly 50pt and asks the provider to load. Because
`LiveAdMobBridge.loadBanner` still throws in v2.3.4, real production
shows the `.failed` caption "Ad unavailable" — honest about the deferred
state, no shimmer/teaser.

## AdMobBannerView stub strategy

A SwiftUI `View` rendering exactly:

```
Text("Ad will load here (v2.3.5)")
    .font(.caption2)
    .foregroundStyle(theme.text.tertiary.resolved)
    .frame(maxWidth: .infinity, maxHeight: .infinity)
```

Sits inside the 50pt rounded rectangle. Tagged
`TODO(v2.3.5): real GADBannerView wiring`. Replacing the body with a
`UIViewRepresentable` / `NSViewRepresentable` wrapping a real
`GADBannerView` is the only change needed at v2.3.5; the surrounding
`BannerSlotView` machinery (gate gating, dismiss, status switch) is
already in place.

Because `LiveAdMobBridge.loadBanner` throws today, production never
reaches the `.loaded` branch — the stub is exercised only by future
recorded ScriptedAdProviderState fixtures (and SwiftUI previews).

## §未決

1. **Snapshot baseline strategy** — the dispatch asks for re-recorded
   HomeView `with-banner` / `without-banner` and BoardView `running-with-banner`
   variants. Because the `BannerSlotView` runs an async `.task` for the
   gate decision and `NSHostingView` snapshot rendering does not pump
   that task before the frame is captured, the recorded image would
   always show the pre-resolution `EmptyView` state. **Proposal**: add a
   second `BannerSlotView` init that accepts an *eager* `initial: Bool`
   for the gate decision (skip the `.task`) and use that init from
   snapshot fixtures only. Left for Leader review — needs a small API
   addition so I held off.
2. **Banner refresh cadence** — `BannerSlotView` calls `refreshBanner()`
   exactly once per mount today. design.md §How.3 mentions "next
   calendar day" as the natural refresh boundary; needs a timer or scene
   phase hook. Deferred to v2.3.7 (boot order) or later.
3. **iPad / macOS layout** — banner is full-width minus 16pt horizontal
   padding on iPhone. On `.regular` size class (iPad / Mac) the
   `BannerSlotView` will stretch across the wider content column; Brand
   audit didn't have an explicit token for max-width. **Proposal**:
   clamp to 320pt on `.regular` so it doesn't sprawl across a 900pt-wide
   Mac window. Deferred.
4. **`@MainActor` on BannerSlotView vs AdGate actor hops** — the slot
   awaits an actor-isolated `AdGate` from the MainActor; under heavy UI
   thrash this could compound hop cost. Not measured. Acceptable for
   v2.3.4 since the gate read happens once per mount; revisit if the
   cadence work in §未決 (2) lands a per-second timer.
