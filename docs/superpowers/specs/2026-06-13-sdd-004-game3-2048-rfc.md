# SDD-004 — Game 3「2048」RFC（v1.0）

- **Status**: RFC_FINAL — 待 user 核准（USER_APPROVED gate）
- **Author**: Leader（草擬）
- **北極星**: 大量共用 modules 推進很多 iOS 小品遊戲，配合廣告跟 IAP（2026-06-13 user 定調）
- **本案定位**: 平台的第一個「scaffold 驗證」遊戲——證明第 N 款遊戲的 per-game 成本曲線

## Why 2048

在「規則核心極小 × 平台形狀零改動」的交集裡最優：

| 平台能力 | 2048 對接方式 |
|---|---|
| DeterminismKit（SplitMix64）| **Seeded Daily**：每日固定方塊生成序列，全球同一局 → Daily Challenge 直接成立 |
| GameCenterKit | 分數排行榜天然適配（Daily 榜 = 種子局分數）；成就掛既有 evaluator |
| PersistenceKit / resume seam | 盤面 + 分數 + 步數 snapshot，照 #478 round-trip 模式；`fetchResume` 注入照 #455 |
| GameAppKit modal flow | 沿用 #474 的 present + `[X]` + Leave Confirmation，零新碼 |
| GameShellKit hubs/completion | Daily/Practice hub shells、completion popup（#480/#481 後的注入式 API）原樣採用 |
| AppMonetizationKit | Banner = Epic 5 的 3 行採用；Remove-Ads IAP 共用 StoreKit2 流程 |
| L10n / Settings / Reminders / Audio | 全平台層，僅補遊戲名詞字串 ×7 locales |

備選曾評估：15-puzzle（規則更小但深度/留存差）、Nonogram（最耐玩但 core ~2k 行，
不符「驗證 scaffold」目的——列為 game 4 候選）。

## Scope（v1.0）

- **玩法**：4×4 經典 2048（滑動合併、2/4 生成、無撤銷——calm brand 傾向「下一局再來」；undo 列 v1.1 候選）
- **模式**：Classic（無限）+ Daily（UTC 種子局，首次嘗試計分，敗局/重玩規則沿用 SDD-003 Epic 8 套餐）
- **Resume**：兩模式皆可恢復（盤面/分數/步數/elapsed）
- **GC**：Daily 分數排行榜 ×1；成就首發 3 個（First 2048 / Daily 完成 / 高磚里程碑），照 #483 net-new 原則不與平台級成就重複
- **Monetization**：banner 全覆蓋（hub/board/settings）+ Remove-Ads IAP，AD-003 同樣適用
- **平台**：iOS + macOS，7 locales，版本號跟平台同步（2.6.0 起跳或獨立 1.0——open question）

## Out of scope（v1.0）

Undo · 多盤面尺寸（5×5）· 主題皮膚 · 對戰/共享 · widget
