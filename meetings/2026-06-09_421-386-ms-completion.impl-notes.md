## цКШшб╖ (Tradeoffs)

- **No elapsed time in the re-view тЖТ `elapsedOverride: "--:--"`** тАФ The shared hero
  always renders an `elapsedLabel`. Passing `elapsedSeconds: 0` would render a
  misleading `0:00` (looks like an instant solve). Options weighed:
  (a) `--:--` placeholder, (b) omit the line, (c) show leaderboard-best as the hero
  time. Picked **(a)**: added an MS-local optional `elapsedOverride: String?` to
  `MinesweeperCompletionView` (does NOT touch the shared `CompletionScreen`). When set,
  it replaces the formatted time label and the a11y interpolation. The leaderboard
  slice still shows the player's RANKED best time тАФ so the actual time IS visible in
  context, just not duplicated in the hero. This is the flagged fork тАФ see ┬зцЬкц▒║.
- **No `Retry` on the re-view (`onRetry: nil`)** тАФ Retry replays the SAME seed as a
  fresh board, which is precisely the "dead replay" #379 set out to avoid for a solved
  daily. The re-view offers only `View leaderboard` + `New Game`. `Retry`'s catalog key
  is still added (the failed-leaderboard block uses it).
