## 設計決定 (Design decisions)

- **Info.plist ATT already present** — Spec §Half B says add `NSUserTrackingUsageDescription`. On inspection `App/Tiles2048/Info.plist` already has the key with 2048-flavored copy. No change needed; the key comment even references an `InfoPlist.xcstrings` file. This is pre-existing correct work; I will verify the xcstrings file exists and has the localized value.

- **MS does not have att.primer.* keys** — Only Sudoku has `att.primer.*` in its Localizable.xcstrings. MS is missing them (it merged #572 without adding them). The spec says 2048 must have them per the gate. I will add all 4 att.primer.* keys + reminder-related keys.

- **LiveRouteFactory for 2048 needs `makeReminderSettings` + `soundPlayer`/`audioSettings`** — The MS LiveRouteFactory wires these into `Game2048SettingsView` (which currently passes `nil` for both). After migration, the `makeGameAppWithDeps` call provides these via GameDeps. The `GameConfig.makeRouteFactory` closure receives the deps bag. I need to add `makeReminderSettings`, `soundPlayer`, `audioSettings` to 2048's `LiveRouteFactory` init and threading — mirroring MS LiveRouteFactory.

- **Game2048SettingsView needs reminder + audio wiring** — Currently passes `nil` to SettingsScreen for both. After the `makeRouteFactory` closure gets deps, the factory will pass them through. The SettingsView itself needs parameters added — mirroring MinesweeperSettingsView which already has these params.

- **Game2048Root.swift will be deleted** — `Game2048AppComposition.rootView` will be replaced by `wired.view`. The per-game `Game2048Root` struct becomes dead code after the migration; the shared `GameRoot` is used via `makeGameAppWithDeps`.

- **Game2048HomeView.swift + Game2048HomeViewModel.swift will be deleted** — Their content is replaced by `GameHomeView` (universal) + `GameConfig.homeModes`. The `homeModes` dict mirrors the former `subtitleKey` literals exactly so behavior is unchanged.

- **`fetchResume` closure form** — In MS pattern: `fetchResume: { _ in { [savedGameStore] in ... } }`. The inner closure captures `savedGameStore`. This moves the `makeFetchResume` logic inline into the closure, making `Live+Resume.swift` deletable.

- **L10n reminder content keys are literal strings** — The reminder copy (`"Today's 2048 Tiles is ready"`, `"Never miss a Daily"`, etc.) are used directly as `LocalizedStringKey` literals in ReminderContent / ReminderPrimerCopy. These must be added to `Localizable.xcstrings` with all 7 locales.

## 偏離 (Deviations)

- **Game2048SettingsView must be extended** — The spec says to wire audio/reminders via the `makeRouteFactory` closure; to do that properly the `Game2048SettingsView` needs `reminderSettings` and `audioSettings` params (currently both nil). I'm adding those params to mirror MS's MinesweeperSettingsView. This is additive, not a spec deviation — it's the necessary plumbing for the settings sections to appear.

- **LiveRouteFactory needs 3 new optional params** — `makeReminderSettings`, `soundPlayer`, `audioSettings` — mirroring MS. The existing `init` uses all optionals so this is backward-compatible; `preview()` still works without them.

## 折衷 (Tradeoffs)

Chose: notification title `"Today's 2048 Tiles is ready"`, body `"Your daily board is waiting — keep coming back."`, primer title `"Never miss a Daily"`, primer lede `"We'll let you know the moment tomorrow's Daily 2048 is ready."`. Mirrors MS shape exactly.

- **att.primer.* key translations** — Reusing Sudoku's translations verbatim (same text, no game-specific customization needed for "Keep your ads relevant").
