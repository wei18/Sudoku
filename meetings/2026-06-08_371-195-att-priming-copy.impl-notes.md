## What I implemented vs proposed

### Implemented (clear/safe)
1. **Localized `NSUserTrackingUsageDescription`** — created `App/Sudoku/Resources/InfoPlist.xcstrings` + `App/Minesweeper/Resources/InfoPlist.xcstrings`, each with the single key fully translated to all 7 locales. Wired both into `Project.swift` `resources:`. Left the existing Info.plist literal in place as the base/fallback (standard Apple pattern — catalog overrides per-locale, literal is the ultimate fallback). Refined the English copy slightly toward the approved "relevance, not a profile" framing.

### Proposed (NOT implemented — needs product decision + new UI surface)
2. **Priming timing fix (#371 defect 1)** — moving the ATT trigger off cold launch requires a NEW priming pre-prompt View + a trigger seam (e.g. fire after first Home appearance, or before the first ad benefits from personalization). No such view exists; building it touches AppComposition boot order + a new SudokuUI/GameShellUI surface. This is more than a surgical edit and the exact trigger is a flagged decision, so I propose the design + copy rather than wiring it blind.
3. **Pre-prompt screen copy (#195)** — drafted full copy (title/body/allow/not-now) in 7 locales, ready to drop into the priming view once the trigger is decided. Kept as a proposal section here (not added to a catalog yet, since the view keys don't exist — adding orphan keys to `Localizable.xcstrings` would have no renderer and would still need to pass the L10n gate with no UI behind them).

## Why I did NOT touch the boot order
- design.md §How.4 wants ATT after Home seen; but the *correct* seam (priming view + first-ad-context vs post-first-game trigger) is the exact product decision the task says to FLAG. Rewiring `bootMonetization()` to split UMP (can stay early) from ATT (must defer) is a real change with a UX dependency. Karpathy §1/§3: surface the tradeoff, make the surgical change only, don't build speculative UI. I leave UMP→ATT→AdMob intact and propose the split.

Refined English (≤ iOS norm, ~135 chars):

(Minesweeper variant swaps the app name.) Full 7-locale table in the two `InfoPlist.xcstrings`.

## Flagged product decisions (Leader/user)
- **F1 — Priming trigger:** cold-launch (current, wrong) vs post-Home-appear vs first-ad-context vs post-first-completed-game. My recommendation: after the user has seen Home AND at the first moment an ad would actually benefit from personalization (lazy, contextual). Needs confirmation before I rewire boot.
- **F2 — Does Minesweeper prompt ATT at all?** Both apps currently do. Option: Sudoku-only ATT (MS keeps non-personalized ads, no prompt) to reduce permission fatigue on the lower-monetization app. Localized string is shipped for both regardless so neither crashes.
- **F3 — Final copy wording/tone** for both the usage description and the pre-prompt screen (drafts above).
- **F4 — Should UMP stay at cold launch** while only ATT defers? (UMP/GDPR consent is arguably fine early; ATT is the one design.md says to defer.)
