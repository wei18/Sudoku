# Tiles2048 migration to GameConfig/makeGameApp + capability-fill (SDD-005 Pillar C, 2048)

**Date:** 2026-06-19 · **Status:** implementing · Parent #479 · Last app to converge.
**Template:** the MS migration (#572, merged) — `MinesweeperAppComposition/Live.swift` + `MinesweeperAppComposition.swift` (which mirror Sudoku #557). 2048 mirrors MS exactly for the composition half.

## Audit
2048 composition (`Game2048AppComposition`: Live 245 / Live+Resume 40 / LiveRouteFactory 241 / struct 102) + UI (`Game2048Root` 80 / RootViewModel 16 / Home View 67 / VM 64) is the same shape as MS pre-migration. **But 2048 has NO audio, NO reminders, NO ATT** (grep confirmed). Since `makeGameApp` wires these UNIVERSALLY (GameConfig requires audio + reminder content), migrating 2048 FILLS these three capability gaps — which is feature work + L10n, not a pure refactor. (2048 is pre-ship/parked #501, so no shipping-user regression risk.)

## Task — two halves

### Half A: composition migration (mirror MS #572)
- Build a `GameConfig<AppRoute>` for 2048 (subsystem `com.wei18.tiles2048`, ckConfig `.tiles2048`, `tiles2048RemoveAdsProductId`, `Game2048Theme()`, title "2048 Tiles", home mode cards from `Game2048HomeViewModel.modeItems`, `fetchResume` ported from `Live+Resume.swift`, `makeRouteFactory` = 2048 `LiveRouteFactory` unchanged, `presentLeaderboard: { GameCenterDashboard.present() }`).
- `Game2048AppComposition.live()` → config + `makeGameAppWithDeps`, mount `wired.view`. Keep public field shape its tests/App read.
- **Delete**: `Live+Resume.swift`, `Game2048Root.swift`, `Game2048Home View.swift` + `Game2048HomeViewModel.swift`. Keep `LiveRouteFactory` (per-game routes; on `boardDestination` from #559) + the `Game2048RootViewModel` typealias (a test reads it).
- 2048 gameplay (tile board, no-mistakes, `Game2048GameViewModel`, daily seed) stays per-game, untouched.

### Half B: capability fill (the gaps makeGameApp forces) + L10n
- **Audio**: `AudioConfig(keyPrefix: "com.wei18.tiles2048.audio")`. makeGameApp wires the audio stack; 2048 has no assets → silent (LiveSoundPlayer tolerates missing). The Settings Sound section appears (acceptable; wiring gameplay cues is a later polish, NOT this PR).
- **Reminders**: `ReminderContentConfig` with 2048 daily-ready copy (mirror Sudoku/MS copy shape, 2048 wording). New `LocalizedStringKey`s.
- **ATT**: makeGameApp wires `ATTPrimerCoordinator` universally (2048 has ads). Needs `att.primer.*` strings + `NSUserTrackingUsageDescription` in 2048's Info.plist.
- **L10n (CI-gated — `mise run scan:l10n` enforces 7 locales)**: author the new reminder + `att.primer.*` keys in `App/Tiles2048/Resources/Localizable.xcstrings` (en source) then translate to the other 6 locales (zh-Hant, zh-Hans, ja, ko, es, th) via the **ai-translated-localization** skill. Plus the `NSUserTrackingUsageDescription` value localized. Reuse Sudoku's att.primer.* + reminder wording as the translation reference (same keys already localized there).
- **Info.plist**: add `NSUserTrackingUsageDescription` to `App/Tiles2048` (follow the existing build-time/Info.plist pattern Sudoku/MS use).
