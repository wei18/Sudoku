## Decisions

### Toast surface

- **Inline Label stayed** in `SettingsView` per dispatch spec. `latestMessage`
  is the a11y / VoiceOver source of truth; the toast is a transient visual
  surface only. VoiceOver does not reliably announce a bottom-overlay toast.
- **Positioning**: bottom-center, 24pt from screen bottom, capsule shape,
  16pt horizontal / 10pt vertical padding. Tint via `theme.status.success`
  / `theme.status.error`. White foreground for AA contrast on both tints.
- **Animation**: ease-in-out 0.2s, move-from-bottom + opacity. No bouncy
  spring (Brand "calm" contract).
- **Auto-dismiss**: per-toast `Duration`, default 3s. `show(_:)` cancels
  any in-flight dismiss task before replacing the current toast, so a
  rapid second toast does not get clobbered by the first's timer.

### purchaseUpdates() subscription

- **Cancellation**: stored as `Task<Void, Never>?` on the controller,
  cancelled by `deinit` and re-armed by every `startListening()` call.
  `bootstrap()` calls `startListening()`; SettingsView + HomeView each
  invoke `bootstrap()` from their `.task`, so a second invocation just
  cancels + re-subscribes (idempotent).
- **Single-consumer constraint**: `FakeIAPClient.purchaseUpdates()` uses
  one `AsyncStream` continuation. Tests cope by not creating their own
  subscriber; the controller is the sole consumer.
- **Test coverage**: `MonetizationStateControllerUpdatesTests` covers
  `.revoked` → flag flip false + failure toast, `.purchased` → flag flip
  true + success toast, unrelated `productId` → ignored. Also covers
  the in-band purchase / restore toast push paths.

  (clean parse).
- Content: one non-consumable, productID `com.wei18.sudoku.iap.remove_ads`,
  $2.99 USD, locales en_US + zh_Hant, family-shareable.
