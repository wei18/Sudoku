## Audit (resolved) — Home is near-identical across all 3 games
All three `*HomeView` are thin wrappers over `GameShellUI.HomeScreen`. Per-game bits only:
- nav title; per-mode subtitle copy; a11y namespace; the per-app `Route` values for daily/practice/settings.
- Sudoku ALSO has: `header` slot (ResumePill, #387) + `attPrimer` (ATT priming on ad context). **MS/2048 lack both = universal-capability gaps (bugs), filled on their migration.**
`HomeViewModel` logic (modeItems from `HomeMode.allCases`, GC-auth-gated `.leaderboard` → `GameCenterDashboard.present()` else signed-out alert, path binding) is identical structure; per-game = subtitle copy + route mapping.

## Target shapes (GameAppKit)
- **`GameHomeViewModel<Route>`** (generic, GameAppKit) — generalize `HomeViewModel`: path binding + `authState` + `showGameCenterSignedOutAlert` binding + `modeItems` built from injected per-game card content + shared `select()` (daily/practice/settings → injected route; leaderboard → GC-auth gate → `GameCenterDashboard.present()` or alert). Derive the exact shape from the 3 existing VMs (`SudokuUI/Home/HomeViewModel`, `MinesweeperUI/Home/MinesweeperHomeViewModel`, `Game2048UI/Home/Game2048HomeViewModel`).
- **`GameHomeView`** (GameAppKit) — generalize the 3 HomeViews: `HomeScreen(items:header:banner:)` + UNIVERSAL ResumePill header (from `rootVM.resumeCandidate` → `resumeTapped`) + UNIVERSAL ATT-primed `BannerSlotView` (deps.adProvider/adGate/attPrimer, themed from `\.theme`) + per-game nav title + a11y namespace. Keeps SudokuUI's `bannerSlot` onAdContext ATT-priming behaviour (universal now).
- **`GameConfig` additions:** per-game home content — mode subtitle copy (per `HomeMode`), route mapping for daily/practice/settings, a11y namespace. (Reuse/extend; keep `title` already present.)
- **`makeGameApp` (MakeGameApp.swift):** build `GameHomeView` as the GameRoot `rootContent`; apply universal `.attPrimerSheet(deps.attPrimer)` + the GC-signed-out `.alert` (bound to `rootVM.showGameCenterSignedOutAlert`) on the returned view. `wired.view` becomes the real mount (no longer dead).

Any moved PNG = behaviour changed → STOP, investigate, do NOT re-record. This is the load-bearing gate.
