## иЁӯиЁҲжұәе®ҡ (Design decisions)

- **Method rename `startListening()` вҶ’ `startListeningForLifetimeOfApp()`** вҖ” RCA `meetings/2026-05-25_swift-test-hang-rca.md В§Fix B` names the new method `startListeningForLifetimeOfApp()`. The existing controller already has a separated `startListening()` private-ish surface but `bootstrap()` calls it eagerly, defeating the split. Renaming to the RCA-prescribed name makes the lifecycle commitment explicit at the call site (`AppComposition.live`) and matches the spec verbatim.

- **Keep `updatesTask?.cancel()` in `deinit`** вҖ” defensive; harmless even if the task was never started (Optional chaining no-ops).

- **Tests use `defer { await iap.finishUpdates() }`** вҖ” RCA prescribes this; `FakeIAPClient.finishUpdates()` is already `public` (confirmed in `Packages/AppMonetizationKit/Sources/MonetizationTesting/FakeIAPClient.swift:59`). Each test that previously called `await controller.bootstrap()` expecting the listener to be active now also calls `controller.startListeningForLifetimeOfApp()` and defers `iap.finishUpdates()`.

- **Non-listener tests untouched** вҖ” The 3 tests `purchaseTap_pushesSuccessToast_whenWired`, `purchaseTap_pushesFailureToast_onFailure`, `restoreTap_pushesSuccessToast_whenWired` don't exercise `purchaseUpdates()` so they don't need the new method nor `finishUpdates()`. Surgical = leave them alone.
