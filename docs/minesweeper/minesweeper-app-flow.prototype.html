New Game (root)

Root content of MinesweeperRoot → GameShellUI.RootShellView → NewGameView. A segmented difficulty Picker (real dims from Difficulty.swift) + a .borderedProminent Start button that does path.append(.board(difficulty, seed)) with a fresh random UInt64 seed per tap. Sidebar (regular size class) holds the two Standard-tier entries: New Game / Settings.

Board

MinesweeperBoardView (wrapped by LiveRouteFactory with a "New Game" toolbar button). Layout top→bottom: status bar (flag count / status / elapsed, 1 Hz TimelineView), the Reveal/Flag mode toggle (segmented Picker — the key Tier-0 #3 affordance), a GeometryReader-scaled grid (32 pt cell floor; Expert 16×30 clamps to the floor and scrolls both axes), a win/lose .regularMaterial overlay, and the banner slot (suppressed during terminal states). Cell glyphs use raw SwiftUI primitives — no theme tokens.

Settings

SettingsView → GameShellUI.SettingsShellView (grouped Form). Three sections: Purchases (shared MonetizationUI Remove Ads / Restore rows — shown only when a controller is wired), About (shared SettingsAboutVersionRow, version from Bundle.main), Storage (shared SettingsStorageSection "Clear cache" → confirmation dialog → real PersistenceProtocol call, parity-only no-op until MS save-flow lands). All tints are .accentColor — MS has no theme.

Design System

As actually rendered by MinesweeperKit — raw SwiftUI primitives.

⚠ No theme system yet. Minesweeper does not have design tokens. Every color below is a raw SwiftUI primitive read straight from the source — covered cells are Color.secondary.opacity(0.25), mines Color.red.opacity(0.6), flags .orange, numbers are the system 1–8 palette, and all chrome tints are .accentColor (system blue). It is not painted in Sudoku's sage palette. Theme extraction is pending — tracked as #278 Tier-1 theme extraction (per MinesweeperBannerSlotView impl-notes D1, reuse blocked on lifting the theme env out of SudokuUI).
