## Architecture chosen

### Trigger seam = BannerSlotView.resolveGateAndLoad() (Sudoku-only)
The cleanest "post-Home + first-ad-context" seam is the Sudoku `BannerSlotView`'s
load path. It runs from `.task` (so the slot — hence Home — is on screen) and only
proceeds when `AdGate.shouldShowBanner == true` (i.e. an ad is actually about to
load = the first moment personalization matters). This is Sudoku-only because
Minesweeper has its own `MinesweeperBannerSlotView` which we do NOT touch.

When the gate is open, BannerSlotView calls `attPrimer?.maybePresentOnAdContext()`
BEFORE loading. The primer:
  - checks ATT status `.notDetermined` (idempotent: skips if already determined)
  - latches `hasOffered` so it only presents once per not-determined lifetime
  - flips `isPrimerPresented = true` → RootView's `.sheet` shows the priming view

### ATTPrimerCoordinator (new, SudokuUI) — mirrors ReminderPrimerCoordinator
`@MainActor @Observable`. Injected with two closures to preserve the AdsAdMob
isolation contract (SudokuUI must NOT import AdsAdMob — only MonetizationCore):
  - `isNotDetermined: @Sendable () async -> Bool`
  - `requestSystemPrompt: @Sendable () async -> Void`
AppComposition wires these to `ATTPresenter` (AppComposition already depends on
AdsAdMob). "Continue" → requestSystemPrompt (system ATT dialog). "Not now" →
dismiss, no request, hasOffered stays true (one offer per launch session).

### Boot change (F4)
`MonetizationBootBridges.live` had `requestATT` firing `ATTPresenter.requestIfNeeded`
at boot. Replace that closure body with a no-op so UMP + AdMob stay and the
coordinator's 3-step shape + ordering tests stay intact. ATT is now driven only by
the primer's "Continue".

### Sudoku-only proof
- ATT request only reachable via `ATTPresenter` wired into the primer closure in
  Sudoku `AppComposition`; the boot coordinator's ATT step is now a no-op.
- MinesweeperAppComposition has NO bootMonetization, NO ATTPresenter reference, and
  its banner is `MinesweeperBannerSlotView` (separate file, no attPrimer param).
- New test asserts MS composition never references / requests ATT.
