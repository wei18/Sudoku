Completion & Leaderboard SHIPPED (#291/#292/#386 · overlay, no best-time row)

Mirror of Sudoku's CompletionView + Game-Center pattern (#49). On a terminal board, MinesweeperCompletionView mounts as an overlay on the live board (#388 — full-cover, not a stack push); re-viewing an already-solved daily uses the standalone .completion route (#386). "View full leaderboard" and the Home Leaderboard card both present Apple's native Game Center dashboard modally (#291) — a side-effect, not a stack push (no .leaderboard route). Divergence from this mock: no best-time row — by design MS stores no completed-game elapsed (the #455 save-flow persists only in-progress boards); the player's ranked time shows in the leaderboard slice instead.

Design System — MS theme IMPLEMENTED (MinesweeperTheme.swift)

These are the shipped token values — MinesweeperUI/Theme/MinesweeperTheme.swift (+ MinesweeperCellTokens), injected at the composition root via @Environment(\.theme) exactly as SudokuUI does. The set mirrors the structure of Sudoku's DefaultTheme.swift (surface / cell / text / accent / status / difficulty, each a light/dark pair) with a distinct identity palette: slate-blueprint blue + signal-orange flag + mine-red, a sibling to Sudoku's sage / warm-paper rather than a clone.

Note: changing these values churns the MS snapshot baselines (the themed board frames are pinned by MinesweeperUITests). Each swatch below shows light · dark.
