Entry & Home

RootView wraps GameShellUI.RootShellView — a NavigationSplitView: on macOS / iPad-regular a sidebar (Daily · Practice · Leaderboard · Settings) sits beside the detail column; on iPhone-compact the sidebar collapses and the detail column shows HomeView directly. The root content is HomeView with an optional ResumePill on top. Navigation is a single path: Binding<[AppRoute]> stack.

Daily

DailyHubView (via DailyHubShellView) lists 3 puzzles for today — one per difficulty, color-dot + completion checkmark + a decorative MiniBoardStrip. When the generator can't produce today's set, Sudoku's own DailyHubState enters .exhausted (which the generic shell maps to its HubLoadState.empty case) and an .alert surfaces.

Practice

PracticeHubView (via PracticeHubShellView): a segmented difficulty Picker tinted with the selected difficulty token, and a "Draw new puzzle" card. The CTA draws then pushes the board in one tap; sub-100ms draws skip indicators, slower draws redact the puzzle-id hint.

Board (gameplay)

BoardLoaderView resolves a puzzleId async (ProgressView), then mounts BoardView: header (difficulty · elapsed timer · pause) over a 9×9 grid (no glass), an optional banner between grid and pad (suppressed while paused), and the DigitPadView — iPhone shows a unified Undo/Redo/Notes/Erase row above a 1×9 digit strip. Pause shows PauseOverlayView (ultraThinMaterial). Mac uses a 2-column layout with a 260pt control rail.

Completion & Leaderboard

On solve, BoardView pushes CompletionView: a "Solved!" hero (elapsed time) plus a leaderboard slice (or sign-in / retry / loading variants). "View full leaderboard" and the Home Leaderboard card both present Apple's native Game Center dashboard modally — there is no .leaderboard AppRoute (removed in issue #49); it's a side-effect, not a stack push.

Settings

SettingsView wraps the shared SettingsUI.SettingsScreen (via SettingsShellView) — a native grouped Form assembling six shared sections in order: Purchases (Remove Ads / Restore, hidden once purchased), Reminders (daily-reminder toggle + fire time), Sound (mute / volumes / BGM / haptics), About (Version + Generator), Notices (links + copyright), and a shared Storage section (Clear cache). Purchase / restore / clear-cache results surface through the bottom-center ToastController mounted on RootView.

macOS & iPad-regular layouts

Same view code, regular size class. NavigationStackHost swaps the iPhone NavigationStack for a NavigationSplitView — a config-driven sidebar (from RootView.sidebarItems) beside a detail pane that hosts the same NavigationStack(path:). Hubs / Settings / Board render in the detail column; BoardView and DigitPadView each branch to a macLayout at sizeClass == .regular. These frames render the real Mac behavior, not a re-imagined design.

Platform-shared surfaces (not drawn as separate Mac frames): Leaderboard is Apple's native Game Center dashboard — presented modally, identical on Mac (issue #49), see S10. The Daily-exhausted .alert (S04), the Board PauseOverlayView (S08), and the BoardLoaderView loader (S06) are size-class-agnostic overlays — the same modifiers in either layout, so they're not re-rendered at Mac scale. Each Mac frame below carries the macOS / iPad-regular share the NavigationSplitView layout labeling (sidebar + detail, same view code, regular size class).

Design System

Tokens read directly from DefaultTheme.swift (v1 ships exactly one Theme) and Theme.swift. Each color is a light/dark pair; the left half of the split swatch is light, the right is dark. SwiftUI access path: @Environment(\.theme) → e.g. theme.accent.primary.resolved.
