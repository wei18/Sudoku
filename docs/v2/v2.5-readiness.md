## Rollback plan

Two kill-switches are available without a code change:

- **Banner crash on real device**: ASC Phased Release pause; rotate AdMob
  ad unit to one that returns 0% fill вҖ” the banner slot will show
  `Ad unavailable` placeholder instead of a crash. If `LiveAdMobBridge`
  itself crashes, ship a hotfix that throws from `start()` (banner slot
  honestly renders `.failed`).
- **IAP failures**: rotate the ASC product to `Removed from Sale` вҖ” the
  Settings + HomeView Remove Ads CTAs will see an empty `availableProducts`
  list and fall back to the `"$2.99"` placeholder, but `purchase()` will
  return `.failed` honestly with a toast.

If a deeper kill-switch is needed, the cleanest seam is
`MonetizationStateController` вҖ” add a `disabled: Bool` flag read from
remote config / env var that short-circuits `purchaseRemoveAds()` and
flips `hasPurchasedRemoveAds = false` so the Settings rows disappear.
Tracked as a future spike, not yet built.

## App Privacy questionnaire (manual)

> **Why manual?** The official App Store Connect REST API has no endpoint
> for the App Privacy questionnaire. Fastlane's `upload_app_privacy_details_to_app_store`
> uses Apple-ID session cookies against undocumented endpoints вҖ” that's a
> security downgrade vs our `.p8` JWT auth, so we fill the form by hand.
> Investigation log: `meetings/2026-05-23_asc-app-privacy-api-research.impl-notes.md`.

### Navigation

1. Sign in to App Store Connect вҶ’ **Apps** вҶ’ **Sudoku** (app ID `6771248206`).
2. Left sidebar вҶ’ **App Privacy** (йҡұз§Ғж¬Ҡ).
3. First time only: click **Get Started** (й–Ӣе§ӢдҪҝз”Ё). On subsequent edits,
   click **Edit** (з·ЁијҜ) next to "Data Types" (иіҮж–ҷйЎһеһӢ).
4. URL pattern: `https://appstoreconnect.apple.com/apps/6771248206/distribution/privacy`
