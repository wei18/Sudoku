The leading XML comment block was rewritten to reflect v2 stance (no own analytics/backend; AdMob does tracking on our behalf subject to ATT + UMP; "Remove Ads" IAP via StoreKit 2 introduces no extra tracking).

### Tracking-domain snapshot provenance

The 8 domains are the standard AdMob ad-serving tracking domains as of 2026-05-21, derived from Google's AdMob privacy manifest documentation (<https://developers.google.com/admob/ios/privacy/manifest>). Review on every Google Mobile Ads SDK version bump — Google may add `google.com`-subdomain endpoints, regional CDNs, or remove sunset domains. Suggested cadence: check on every SDK pin change in `Packages/AppMonetizationKit/Package.swift`.

### Required-Reason API rationale

Only `UserDefaults` (CA92.1 — "Access info from same app, per documentation") is declared because the AdMob SDK uses `UserDefaults` internally for consent state, frequency capping, and SDK config. We deliberately do not declare `FileTimestamp`, `DiskSpace`, or `SystemBootTime` since Sudoku's own code does not access them; if a future Google SDK build starts using those, this manifest will need to grow.
