## Rollback plan

Two kill-switches are available without a code change:

- **Banner crash on real device**: ASC Phased Release pause; rotate AdMob
  ad unit to one that returns 0% fill — the banner slot will show
  `Ad unavailable` placeholder instead of a crash. If `LiveAdMobBridge`
  itself crashes, ship a hotfix that throws from `start()` (banner slot
  honestly renders `.failed`).
- **IAP failures**: rotate the ASC product to `Removed from Sale` — the
  Settings + HomeView Remove Ads CTAs will see an empty `availableProducts`
  list and fall back to the `"$2.99"` placeholder, but `purchase()` will
  return `.failed` honestly with a toast.

If a deeper kill-switch is needed, the cleanest seam is
`MonetizationStateController` — add a `disabled: Bool` flag read from
remote config / env var that short-circuits `purchaseRemoveAds()` and
flips `hasPurchasedRemoveAds = false` so the Settings rows disappear.
Tracked as a future spike, not yet built.
